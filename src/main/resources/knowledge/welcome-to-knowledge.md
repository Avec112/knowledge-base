# Welcome to KnowledgeBase

Welcome to your organization's knowledge repository! This is your central hub for documentation, guides, and shared knowledge.

## What is KnowledgeBase?

KnowledgeBase is an integrated wiki system designed to help teams collaborate and share information effectively. Write articles using simple Markdown syntax and organize your knowledge in one place.

## Getting Started

### For Readers
- Browse articles from the list on the left
- Click on any article to read its content
- All articles are rendered from Markdown for easy reading

### For Administrators
As an administrator, you have additional capabilities:
- **Create** new articles using the "Create" button
- **Edit** existing articles with the "Edit" button
- **Delete** articles that are no longer needed
- All changes are saved automatically

## Markdown Syntax Guide

Here's a quick reference for writing in Markdown:

### Headings
```markdown
# Heading 1
## Heading 2
### Heading 3
```

### Text Formatting
- **Bold text**: `**bold text**`
- *Italic text*: `*italic text*`
- ~~Strikethrough~~: `~~strikethrough~~`

### Lists

**Unordered list:**
Use `-` or `*` for unordered lists.
```markdown
- Item 1
- Item 2
  - Nested item
  - Another nested item
```
- Item 1
- Item 2
  - Nested item
  - Another nested item

**Ordered list:**
```markdown
1. First item
2. Second item
3. Third item
```
1. First item
2. Second item
3. Third item

### Links and Images
- Link: `[Link text](https://example.com)`
- Image: `![Alt text](image-url.jpg)`

### Code

Inline code: `code here`

Code block:
```java
public class Example {
    public static void main(String[] args) {
        System.out.println("Hello, KnowledgeBase!");
    }
}
```

### Quotes
```markdown
> This is a blockquote.
> It can span multiple lines.
``` 
> This is a blockquote.
> It can span multiple lines.

## Best Practices

1. **Use descriptive titles** - Make it easy to find articles
2. **Organize with headings** - Structure your content clearly
3. **Keep it updated** - Review and update articles regularly
4. **Link between articles** - Create a web of knowledge
5. **Use code blocks** - Share code examples when relevant
